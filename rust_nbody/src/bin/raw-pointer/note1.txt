Rust 原始指针（Raw Pointer）超通俗解释
我用最简单、最不绕的方式给你讲清楚 Rust 原始指针是什么、为什么存在、怎么用、什么时候用。
一、先一句话总结
Rust 原始指针 = 不受 Rust 安全规则保护的 “裸指针”
它就是一个内存地址，和 C 语言的指针几乎一模一样
Rust 不会帮你检查空指针、野指针、重复释放、数据竞争
必须写在 unsafe 块里才能使用
二、两种原始指针
Rust 有两种原始指针，写法很直观：
rust
运行
*const T    // 只读原始指针（不能改指向的值）
*mut T      // 可写原始指针（能改指向的值）
对比：
&T = 安全共享引用（Rust 严格检查）
&mut T= 安全可变引用（Rust 严格检查）
*const T / *mut T = 裸奔的指针（不检查）
三、怎么创建原始指针？
非常简单，直接从安全引用转过去就行：
rust
运行
let mut x = 10;

// 从普通引用转成原始指针
let p1: *const i32 = &x;    // 只读指针
let p2: *mut i32 = &mut x;  // 可写指针
这一步是安全的！
只有 ** 解引用（读取 / 修改指针指向的值）** 才必须进 unsafe。
四、怎么使用原始指针？（必须 unsafe）
rust
运行
let mut x = 10;
let p = &mut x as *mut i32;  // 转成原始指针

// 必须用 unsafe 才能读写指针
unsafe {
    *p = 20;  // 修改指针指向的值
    println!("{}", *p); // 20
}

println!("{}", x); // 20（确实改成功了）
unsafe 的意思不是 “危险代码”，而是：
“这段代码的安全性，我自己保证，不用 Rust 检查”
五、原始指针有什么特点？（重点）
原始指针不受 Rust 三大安全规则限制：
可以是空指针
rust
运行
let p: *const i32 = std::ptr::null();
可以指向无效内存
多个指针同时指向同一个值（不检查共享）
不自动释放内存
跨线程使用不检查数据竞争
一句话：用起来像 C 指针，自由但危险。
六、为什么 Rust 要有原始指针？
既然危险，为什么还要存在？
因为：
写 FFI（调用 C/C++ 代码）必须用
实现安全抽象（比如 Vec、String、Box 底层）
高性能场景：绕过安全检查提升速度
嵌入式、操作系统内核开发
99% 的普通 Rust 开发根本用不到原始指针！
只有写底层库、系统编程、FFI 时才会用。
七、和安全引用的区别（最关键）
表格
特性	安全引用 &T / &mut T	原始指针 *const T / *mut T
Rust 安全检查	✅ 强制	❌ 不检查
空指针	❌ 不可能	✅ 允许
多个可变引用	❌ 不允许	✅ 允许
使用是否需要 unsafe	❌ 不需要	✅ 必须
用途	日常 99% 开发	底层、FFI、高性能、数据结构
八、最简单的记忆口诀
&T = 安全的共享
&mut T = 安全的修改
*const T / *mut T = 裸奔指针，自己负责安全
用必须写 unsafe
总结
原始指针就是内存地址，和 C 指针一样
分 *const T（只读）和 *mut T（可写）
创建安全，使用必须 unsafe
Rust 不提供任何安全保障
用于 FFI、底层数据结构、系统编程
普通业务代码永远不需要用
