Rust 原始指针 vs 智能指针（Box / Arc）核心区别
先把三类角色定死：
原始指针：*const T / *mut T
智能指针：Box<T>、Arc<T>、Rc<T> 这类
一、先一句话定性
原始指针：就是裸内存地址，无任何安全保障、无所有权、无自动内存管理，全靠程序员自己兜底，必须 unsafe 操作。
智能指针：是封装了原始指针的安全结构体，在安全 Rust 里就能用，自带所有权、自动析构、借用规则、生命周期约束，不用手动管内存。
二、核心维度对比表
表格
对比维度	原始指针 *const/*mut	Box<T>	Arc<T>
是否安全	不安全，需 unsafe	完全安全，无需 unsafe	完全安全，无需 unsafe
内存管理	手动，不自动释放	独占所有权，离开作用域自动释放堆内存	引用计数，计数为 0 自动释放
借用规则	完全绕过 Rust 借用检查	遵循严格借用规则：同一时刻要么只读、要么独占写	遵循借用规则，多线程安全共享
生命周期	无生命周期约束，极易野指针	受生命周期、所有权严格约束	不受短期生命周期束缚，可跨作用域共享
线程安全	非 Send/Sync，多线程直接用必数据竞争	默认 Send，非 Sync	原子引用计数，多线程安全
用途定位	FFI、底层数据结构、内核、寄存器访问	堆上存数据、递归结构（链表 / 树）、转移所有权	多线程共享只读数据
引用计数	无	无	有（原子计数器）
空值允许	允许空指针、野指针	绝不为空	绝不为空
三、逐个拆解关键差异
1. 所有权与内存自动回收
原始指针
不拥有任何数据所有权，只是个地址；指向的内存什么时候释放、是否已经销毁，完全不管，很容易悬垂指针。
Box<T>
独占堆内存所有权，变量失效时自动 drop 释放堆内存，不用手动 free。
Arc<T>
共享所有权，每克隆一次引用计数 + 1，每销毁一次计数 - 1，计数归 0 自动释放。
2. 安全与 Unsafe
原始指针：解引用、读写、偏移全部要 unsafe，Rust 不做任何检查。
Box / Arc：正常读写、赋值、克隆全在安全代码里，安全检查编译器帮你做完了。
3. 借用规则约束
原始指针：可以随便多个可变指针同时指向同一块内存，直接违反 Rust 内存安全，产生 UB。
Box：严格遵守「不可同时多可变引用」。
Arc：配合 Mutex/RwLock 就能安全多线程共享修改，底层帮你保证安全。
4. 能否为空
原始指针：可以 ptr::null()，随便空，解引用直接崩溃。
Box/Arc：合法状态下永远不为空，编译器保证。
四、使用场景直观区分
用原始指针的场景
FFI 调用 C 接口
手写底层容器、内存池
嵌入式直接操作硬件寄存器固定地址
实现自引用结构底层
用 Box<T> 的场景
把数据放到堆上
递归类型（比如二叉树、链表，避免栈无限递归）
转移大块数据所有权，避免栈拷贝
用 Arc<T> 的场景
多线程之间共享同一份只读数据
线程间传递资源所有权
五、最通俗总结
原始指针：C 风格裸指针，自由、危险、手动管安全和内存，只给底层开发用。
Box：最简单的独占式智能指针，堆内存自动释放，安全替代普通裸指针独占场景。
Arc：带原子引用计数的智能指针，专门做多线程安全共享。
日常业务开发：永远优先用引用、Box、Arc、Rc、Mutex，绝不主动用原始指针。
